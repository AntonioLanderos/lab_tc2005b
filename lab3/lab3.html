Preguntas a responder

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. Internet es una red de computadoras alrededor de todo el mundo conectadas entre sí. En cambio, la web es una colección de páginas que se asienta sobre esa red de computadoras

  2. ¿Cuáles son las partes de un URL?

    3. Una URL consta del protocolo, el nombre del dominio, la extensión del dominio, la ruta al recurso y los parámetros

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
    • HEAD: Pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
    • POST: Se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
    • PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
    • PATCH: Es utilizado para aplicar modificaciones parciales a un recurso
    • DELETE: Borra un recurso en específico

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. El método POST se debe utilizar al enviar un formulario HTML, como cuando ingresas tu usuario y contraseña en algún sitio. Esto se debe a que el método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. El método GET se utiliza cuando a través de un navegador web se accede a una página a través de un URL

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. El código de respuesta de estado satisfactorio HTTP 200 OK indica que la solicitud ha tenido éxito

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. Los errores 404 suelen ser el resultado de enlaces rotos o páginas que se han eliminado, y es tarea del desarrollador asegurarse de que todos los enlaces del sitio web estén funcionando correctamente

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. Los errores 500 son errores del servidor, y pueden ser el resultado de problemas con el software del servidor o con el código del sitio web

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Un atributo HTML5 está depreciado o desaprobado cuando ya no se recomienda su uso en el desarrollo web porque ha sido reemplazado por una mejor alternativa. Por ejemplo: 'center', 'font', y 'strike', entre otros

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    11. HTML5 permite la creación de nuevos elementos y atributos, maneja mejor los elementos multimedia, y tiene una estructura más flexible que HTML4

  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    12. Una tabla en HTML se compone de varios elementos para su estructura, incluyendo 'table' para crear la tabla, 'tr' para las filas, 'td' para las celdas de datos, y 'th' para las celdas de encabezado. Para el estilo, se pueden usar atributos como border para los bordes, padding para el espacio interior de las celdas, y width y height para el ancho y alto de la tabla, entre otros.

  12. ¿Cuáles son los principales controles de una forma HTML5?

    13. Algunos de los principales controles de un formulario en HTML5 incluyen 'input' para campos de entrada, 'textarea' para áreas de texto, 'button' para botones, 'select' para listas desplegables, y 'option' para las opciones en las listas desplegables, entre otros.
